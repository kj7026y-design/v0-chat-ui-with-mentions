# 멘션·캐릭터 대사 삽입 QA/AC

## 자동화 AC

1. `@캐릭터` 입력은 화면/재시도용 ID와 모델용 이름을 함께 보존한다.
2. `ⓣ캐릭터: 대사`는 사용자가 대신 작성한 캐릭터의 확정 발화로 전달한다.
3. 일반 사용자 입력, 캐릭터 대사, 후속 사용자 행동을 한 번에 보내도 시간 순서를 유지한다.
4. 내용이 비어 있는 캐릭터 대사는 생성 요청 전에 차단한다.
5. 자동 진행은 직전 멘션이나 캐릭터 대사를 새로운 사용자 입력으로 복사하지 않고, 사용자 반응을 기다리는 데 멈추지 않은 채 독립적인 다음 사건을 만든다.
6. 저장·복원 뒤 재시도와 재생성에서도 멘션 ID/이름 및 캐릭터 화자를 유지한다.
7. 자동 진행 응답이 최대 분량을 넘으면 중간 문장을 자르지 않고 완결된 문장 경계에서 보정한다.
8. 입력 블록 분리 이후에도 텍스트 없이 이미지만 보내는 기존 전송 동작을 유지한다.

실행 명령:

```powershell
npx tsx --test tests/chat-interactions.test.ts
```

## 수동 통합 QA

- 일반 입력: `@강태현 지금 내 말 듣고 있어?` 전송 후 강태현을 수신 대상으로 삼아 바로 반응하는지 확인한다.
- 캐릭터 대사: `ⓣ이서윤: 이제야 왔어?` 전송 후 해당 대사를 반복하지 않고 강태현의 다음 반응부터 이어지는지 확인한다.
- 혼합 입력: 사용자 대사, 빈 줄 두 번, 캐릭터 대사, 빈 줄 두 번, 사용자 지문을 전송해 세 장면 블록 순서가 유지되는지 확인한다.
- 자동 진행: 답변 뒤 입력창을 비운 채 자동 진행해 새 사용자 말이나 이전 멘션이 만들어지지 않고, 캐릭터의 새 행동·결정·정보가 실제로 하나 이상 진행되는지 확인한다.
- 자동 진행 재생성: 자동 진행 답변을 재생성해 자동 진행 상태가 유지되고 폐기 답변을 복사하지 않는지 확인한다.
- 실패 재시도: 네트워크 실패 메시지를 재시도해 원래 멘션 대상과 캐릭터 대사 화자가 유지되는지 확인한다.
